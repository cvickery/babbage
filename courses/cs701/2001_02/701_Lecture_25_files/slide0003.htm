Asynchronous Commands
• Syntax
– & at end of command
• Implementation
– Shell issues prompt for next command without waiting for child
process to finish.
– But must avoid “zombies.”
– Must return exit status to user.
– May be handled by calling waitpid() just before issuing prompts.
•  waitpid() can be coded so it doesn’t block (WNOHANG)
