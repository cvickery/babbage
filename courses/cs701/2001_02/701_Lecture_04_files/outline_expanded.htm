Building an Executable File
Compiler driver manages a pipeline of commands.
Preprocess (cpp)
Compile to to assembly language (cc1 or cc1plus)
Assemble to an object module (as)
Link object modules and libraries (ld)
File suffixes ( .c and .o in particular ) tell the compiler driver where to start in the pipeline.
Some options tell where to stop the pipeline.
-E (stop after cpp); -S (stop after c1); -c (stop after as)
Other important options are passed to commands in the pipeline.
-I, -D (passed to cpp)
-o, -l, -L (passed to ld)

The Make Utility

Built-in Rules
Based on file suffixes.  It knows how to convert .c files into executable files, but doesn’t know how to make .c files, because people do that using an editor.
Environment Variables
You have to set the CC and CFLAGS environment variables to get make to generate the correct gcc command for you.
 setenv CC gcc
 setenv CFLAGS “-g –Wall”
