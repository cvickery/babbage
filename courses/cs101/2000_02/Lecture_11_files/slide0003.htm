Names for JVM Memory
Regions
• Heap
– Holds objects.
– Instance variables are inside objects, so instance variables are in
the heap.
– Holds class “objects.”  (There is more to this than we will go into.)
• Method Area
– Holds the executable instructions (bytecodes) for all methods,
whether static or instance.
• Stack Area
– Holds formal parameters, local variables, and block-local
variables.
– Each thread of execution gets its own stack.
