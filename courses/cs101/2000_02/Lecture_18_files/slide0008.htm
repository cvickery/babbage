Adapter Classes
• Standard libraries use interfaces to define listeners.
– But many applications end up having to provide “nop” (do-
nothing) implementations for most of the methods in the interface.
• Adapter classes implement listener interfaces by providing
nop implementations for all methods in the interface.
– Then you subclass the adapter class and override the methods that
you want to have actually do something.
• Note: There is only one method declared in the
ActionListener interface, so it wouldn’t make sense to have
an “ActionAdapter” class.
