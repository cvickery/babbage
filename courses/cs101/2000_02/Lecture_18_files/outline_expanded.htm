GUI Development
Need a window (JFrame) to hold the components that make up the GUI.
Some components are containers, which hold other components.
JFrame and JPanel are examples of containers.
Other components (controls) provide the visual elements of the graphical user interface.
JLabel is a passive component that can display text.
JButton is an active component that the user can interact with (by pressing it).

Make the Close Button Work
Need a Window Listener
A Window Listener is an instance (object) of any class that implements the java.awt.event.WindowListener interface.
Methods declared in java.awt.event.WindowListener:
void windowActivated( WindowEvent we );
void windowClosed( WindowEvent we );
void windowClosing( WindowEvent we );
void windowDeactivated( WindowEvent we );
void windowDeiconified( WindowEvent we );
void windowIconified( WindowEvent we );
void windowOpened( WindowEvent we );
[ WindowListener_ex.java ]

Adapter Classes
Standard libraries use interfaces to define listeners.
But many applications end up having to provide “nop” (do-nothing) implementations for most of the methods in the interface.
Adapter classes implement listener interfaces by providing nop implementations for all methods in the interface.
Then you subclass the adapter class and override the methods that you want to have actually do something.
Note: There is only one method declared in the ActionListener interface, so it wouldn’t make sense to have an “ActionAdapter” class.

Anonymous Classes
Small classes with no name (the compiler provides the name for you).  Defined on an as-needed basis inside other classes.
Example: Define a subclass of WindowAdapter that overrides the windowClosing() method.
WindowAdapter wa = new WindowAdapter( )
{ /* class definition starts here */
    public void windowClosing( WindowEvent we )
    { /* method definition would go here */ }
 }
