File Operations
(From Chapter 17)

A File Contains a Sequence of Bytes
Java calls this a “stream.”
Most file reads and writes are done sequentially.  The “system” (Java and the operating system) keeps track of the current position in the file so sequential reads/writes move through the file.
Random access is possible.
Involves skipping directly to a particular position in the file.

Classes for File I/O
For character streams (“text” files)

FileWriter
For byte streams (“binary” files)
