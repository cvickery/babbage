jar
• jar [ctx]vf <jar file name> <files>
– Specify exactly one of c, t, x.
• c Create a jar file
• t Test (list the contents of) a jar file
• x Extract files from a jar file.
– v produces “verbose” (wordy) output.
– f says that the next thing in the command is the
name of the jar file
• Jar files are really zip files.  One or more
files compressed and put into a single file.
