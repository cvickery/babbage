MMJB Revisited
Note that the tag editor is just one part of this big application.
We will be developing a stand-alone tag editor patterned on MMJB’s.
But first …

Design Decision
Allow the user to edit multiple files at once.
Each one gets its own window
Need to keep track of how many there are.
For positioning new ones
To know when to exit the application
Don’t need to keep track of open state.
Each window deals with one open file
Dispose of the window when closing the file.

Development Plan
Write an application that writes a message to a log file every time the user interacts with it.
Think of this as a log file editor, where the program writes the text lines instead of the user.
Add user interface components to this application, and make sure each one “works.”
The components will write log file message for now.
In the next step they will deal with MP3 tags.
Convert the application to an editor for MP3 tags.
