OOP Principles
Encapsulation
• Programs = Algorithms + Data Structures
– Use classes to put data and the algorithms (methods)
that manipulate those data together.
• Example: You can use class Vector without
knowing how it is implemented internally.
– Public methods provide the “interface” to the class.
– As long as you don’t change the signatures of the
public methods, you can change the internal details all
you want.
• Benefits: Modularity, Maintainability
