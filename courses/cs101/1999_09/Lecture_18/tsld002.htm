What I Learned about class File

  • getCanonicalPath() removes and references to dot ( . ) or dot-dot ( .. ) in a pathname, but may throw an IOException.

    • Useful for cleaning up what the user types.

  • If you create a File without a complete pathname, getAbsolutePath() assumes it is in the current directory. Use the constructor, File ( File dir, String name ).

    • Needed for argv[0], and inside FileIterator, but JFileChooser returns a complete pathname. (Complete, but not necessarily canonical.)
