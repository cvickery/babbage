More How Many?

  • How many copies of a class’ instance fields are there?

    • One per instance (duh!). May be zero, one, or more.
      • Zero if the only reason for loading the class is to use its static fields and/or methods.
      • One (a “singleton class”) if you need a single object.
      • Multiple (a “template class”) if you need several objects of the same type, but each with different properties (i.e., values of fields.)

  • How many instances of JFileChooser in Project 1?

    • One per iteration of the main loop.
      • At least one; then it depends on what the user does.
