Command Line Arguments

  • If two or more, it’s an error.

  • If one, see whether it’s the name of a file or not. If not, create and initialize it. If it is a file, check to be sure it’s a valid library file.

  • If there is no file (either because there was none on the command line or because the one on the command line was invalid), prompt the user to select/name one.
