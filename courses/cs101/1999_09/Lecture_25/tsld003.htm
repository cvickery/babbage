Debugging

  • Syntax Errors are detected by the compiler.

    • Use the error message to figure out what is wrong
    • Sometimes you have to look a line or two above the one where the error was detected.
    • This is not debugging.

  • Run-time errors require debugging.

    • The problem is to locate the error. Then you should be able to figure out how to fix it.
      • If there is an error message and/or a stack trace, the file and line number should be given.
        • Be sure to set JAVA_COMPILER=NONE
      • Use System.err.println() to figure out where the program goes wrong and to see values of variables.
        • Start them in column 1 to make removal easier

    “Every bug is a puzzle to solve and a chance to learn something new!”

    • Use an interactive debugger if you have an IDE.
      • Avoids the need to insert/remove print statements
