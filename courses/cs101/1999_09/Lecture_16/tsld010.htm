Anonymous Classes

  • Useful when you need to define a very simple class that you will not refer to again.

    • For example, subclasses of adapters where the method(s) being overridden are few and simple.
    • May appear almost anywhere. Such as the argument passed to addWindowListener().

  • Syntax

    new Name() { < class definition > }

    • Name is the name of the class being extended (e.g., the name of an adapter class)
    • Put the definition(s) of the method(s) you want to override inside the curly braces.
    • The anonymous class’ name will be the name of the enclosing class, a dollar sign, and a number. (e.g., LibraryManager$1)
