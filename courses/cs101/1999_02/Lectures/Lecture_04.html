Javadoc and Arrays

Javadoc

Javadoc, which comes as part of Sun's JDK, is a program that generates Web pages that document your code for you. To use javadoc, you must put comments in your code in a certain format. Then you run javadoc with your .java file names as parameters on the command line, and javadoc will generate a set of web pages for you.

The Javadoc command line

In addition to a list of the .java files that you want to document, you need to specify two switches to get javadoc to work properly for you. The first switch is -author, which tells javadoc not to ignore @author tags in your code. (For some reason the default behavior is to ignore them.) The second is the -d switch, which you use to specify a directory where the documentation files are to be put. If you don't use this switch, all the documentation files will go in your current directory, possibly destroying some files you might already have with the same names, and certainly making it difficult to manage the .java and .class files in the directory. The directory to receive the documentation must already exist before you run javadoc.

So, if you have created a sub-directory named HTML to hold the files generated by javadoc, you could run the following command to generate web page documentation files for all the java files in the current directory:

	javadoc -author -d HTML *.java
