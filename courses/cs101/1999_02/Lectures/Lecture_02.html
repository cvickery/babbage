Working with the DOS Command Line

Terminology

  • Folder and Directory generally mean the same thing
  • There is a Current Directory; its pathname appears in the prompt string
    • If you don't give an explicit pathname, the current directory is where files will be saved.
    • On some systems, the command prompt window starts with C:\Windows as the current directory. This is always inappropriate, and you much change it before saving any files. There is a free "Powertoy" called doshere available from Microsoft that lets you right-click on any folder and start a DOS prompt window with that directory as the initial current directory. ([ click here ] to download it.)
  • A "DOS Prompt" is a window that runs a program called command.com (Windows 95 or 98) or cmd.exe (Windows NT or 2000). This program:
    1. Displays a prompt string telling you it is ready for you to type a command line
    2. Reads the command line you type
    3. Executes the command line you typed either by interpreting it directly (a built-in command) or by loading an executable file for execution (an external command)
    4. The program that executes, whether it is a built-in command or an external command, may (or may not) read things you type and write things for you to read in the DOS Prompt window. When the program completes, command.com (cmd.exe) displays another prompt for you to enter a command line.

Command Line Syntax

<command name> [ switches ] [parameters]
    • The <command name> must be any one of the following:
    • The name of a built-in command that command.com interprets itself.
      • dir (list file and directory names), copy (copy files from one directory to another), del (delete files), md (make a directory), rd (remove a directory), cd (change the current directory), set (list environment variable names and values).
    • The exact pathname of an executable file. The file name must end in .exe, .com, or .bat, but the extension does not have to be typed.
      • C:\jdk\1.1.7\bin\java
      • C:\windows\command\xcopy32
    • The name of an executable file that is in one of the directories listed in the PATH environment variable.
      • java
    • The square brackets ( [ … ] ) around switches means this part of the command line is optional. If there are any switches, they begin with either a slash ( / ) or a dash ( - ), depending on the command. Some commands allow switches to be given following, or intermixed with, parameters.
      • java -nojit …
      • C:\windows\xcopy32 *.* /s A:
        • (/s tells xcopy32 to copy subdirectories)
    • The parameters are whatever the command needs to know in order to do its work. Often, they are the names of files.
      • javac MyClass.java
      • C:\"Program Files\pfe\pfe32" Exercise_1.java
        • (Note the use of quotes because of the space in the pathname.)

Environment Variables

Environment variables are name-value pairs that command.com and other programs use to manage housekeeping information.

  • For example, command.com uses the variable named PATH to tell it what directories to search when looking for an executable file. The value of PATH is a semicolon ( ; ) separated list of directories.
  • For example, javac and java use the variable named CLASSPATH to tell them where to find class files. The value of CLASSPATH is a semicolon-separated list of directories, jar files, and zip files that contain class files.
  • To assign a value to an environment variable, type set, the name, an equal sign ( = ), and the value. Don't put any spaces anywhere, even around the equal sign. To set the names and values of all your environment variables, type set. If a variable is already set and you just want to add something to its value, you can refer to the old value by putting the name inside percent ( % ) signs.
    • set PATH=C:\jdk1.1.7\bin;%PATH%
    • Note: PATH is special. You can assign an initial value to it by typing "PATH x;y;z" (where x, y, and z are pathnames of directories) and you can see its current value by using PATH as if it were the name of a command.
    • Commands to assign a value to your PATH are usually put inside C:\autoexec.bat. This is a "batch file" that contains command lines that are automatically executed whenever Windows starts running. (In NT this is done in the System/Environment panel of the Control Panel.)
    • Note: Java does not need to have CLASSPATH set in order to work properly. But if it is set it must list all the places that class files might be found.
