Review of CS-95

Java the Language

  • Relative of C and C++
  • Similar data types, but cleaned up
  • Similar control structures
  • But Memory Management is automatic
  • Portable (not Wintel)
  • International (Unicode characters instead of ASCII or even ISO Latin-1)
  • Web Compatible
  • Applets and Browsers
  • Networking APIs provided: Sockets, Remote Method Invocation (RMI, the ability for a method running on one computer to invoke a method running on another computer), Object Serialization (Objects can be sent over the network.)
  • Other APIs provided with the language: Abstract Windowing Toolkit (AWT, for graphics), Beans (Components that can be used in different applications/applets), Swing (LIke AWT, but user interface elements look like platform-dependent components.)

Java Code Development

  • Compile .java source files to .class files. One source file may generate multiple class files. Source file name must match public class name.
  • Load class files into Java Virtual Machine (JVM): Web Browser, appletviewer, or java, for example.
  • JVM (software) interprets class file byte code.
  • Parts of JVM could be built in hardware, but security, i/o, and memory management would need software support, like an operating system.

Conventional Code Development

  • Compile .cpp source files to .obj object files. Always exactly one-to-one match.
  • Link object modules plus library files (collections of object modules) to produce .exe file.
  • Load/execute executable file. May link with dynamic libraries (.dll files) at this time.
  • CPU (hardware) executes machine language.
  • Example: java.exe, the Java Virtual Machine (!)
