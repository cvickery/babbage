Code Documentation (Comments)

The purpose of the comments is to help the next programmer to find his or her way around in the code easily. Remember, users never see the comments, so information for users doesn't go here.

Each .java file starts with a File Header, which is a set of // comment lines that include:

  • Name of the file
  • Contents of file. (What classes, etc.)
  • @author
Each class and method definition gets its own heading, which looks like this: 
    //  <class or method name>
    //  -----------------------------
    /**
     *  <Description of the class or method>
     */
  • Indentation - Use 2 or three spaces per indentation level. Don't use tab characters (most editors can be set to expand tabs to spaces automatically).
  • Line Width - All lines less than 72 columns wide.
  • Spacing - Follow textbook guidelines. (One space on each side of operators, blank line before and after if, while, etc.)

Recursion

A recursive method calls itself either directly or indirectly. Many problems can be solved efficiently and elegantly using recursion, but the Fibonacci program on page 197 is not one of them!
Fibonacci number Number of Calls Ratio
0 1  
1 1 1.0
2 3 3.0

31 4,356,617 1.618

We say that the complexity of the computation grows exponentially. The number of calls is approximately proportional to 1.618 raised to the Fibonacci number being computed. "Big O" notation is used to represent the "order of magnitude" of the size of a computation. It's O(1.618n) for recursive computation of Fibonacci numbers.

Iterative Fibonacci

Here's pseudocode for a method that computes a Fibonacci number iteratively: 
  long Fibonacci( int n ) {
    if ( n < 0 ) return -1   // error condition
    if ( n < 2 ) return  n   // base conditions
    fBack2 = 0; fBack1 = 1
    for (int i = 0; i < n; i++) {
      f = fBack1 + fBack2
      fBack2 = fBack1
      fBack1 = f
      }
    return f
    }
Try it and compare it's speed with the recursive program on page 197!

Array Objects

Syntax 
    int x[] = new int[5];

    int x[];
    x = new int[5];

    Generalize "int" to any primitive type or class.
x.length is an instance variable -- there is a different one for each Array object.

It's also an example of a final variable -- it's value can't be changed.

Search an Array

  • Linear Search - O(n)
  • Binary Search - O(log(n))

Doubly-Subscripted Array (Matrix)

  • Each row same size: int b[][] = new int[3][3]
  • Each row different: int b[][] = new int[2][]
    b[0] = new int[5]
    b[1] = new int[3]
"Ragged Array" = different size rows.
