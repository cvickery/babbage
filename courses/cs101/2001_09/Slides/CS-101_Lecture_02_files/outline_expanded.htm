The jar Command
Can be used to manage “zip” files.
Jar is a free alternative to WinZip that comes with the JDK.  It’s a command, just like javac and java.
Jar files conventionally have names ending in “.jar,” but you can use “.zip” instead if you want to
Jar files automatically include some additional files that zip files don’t have  (Manifest, etc.), but they can be ignored.
JVMs know how to extract class files from jar files automatically, but for this course you will put just .java files inside the jar files you submit to me.

Jar Command Options
Create a jar file
 jar cvf Assignment_2.jar *.java
Creates a jar file named Assignment_2 that contains all the files that have names ending in .java in it.
List the contents of a jar file
 jar tvf Assignment_2.jar
Lists all the files in the jar file named Assignment_2.jar
Extract files from a jar file
 jar xvf Assignment_2.jar
Extracts all the files from Assignment_2.jar into the current directory.
I will use this command when grading your assignments.
See the online documentation for more details about the options.

HelloCL.java
Like HelloGUI.java, but the user’s name is taken from the Java command line.
 public static void main( String[] args ) …
 args is an array of strings, one for each word typed on the command line by the user.
The elements of an array are accessed by an integer subscript.  The variable names i, j, k are conventional names for subscripts, but any variables can be used.
The number of elements in an array is given by the value of the field named length.
The first element of an array has a subscript value of zero.
 To print each command line argument, if there are any:
     for (int i = 0; i < args.length; i++)
          System.out.print( “ “ + args[i] );
