Object Based Programming
Objects are data structures and the methods that operate on them.
Need to understand the architecture of a computer to understand how objects are implemented in the memory of a computer.

Computers and JVMs
Computer: Processor, Memory, I/O
Pentium is an example of a processor (CPU), but there are many others, each with its own machine language
“Memory” is often called “RAM.”
Holds programs as they are executed by the processor.
Volatile: information lost when electricity goes off.
I/O includes disks, keyboard, mouse, monitor
Java Virtual Machine (JVM)
A program that simulates the behavior of a particular processor.  The machine language is called Java bytecode.
Classfiles contain the Java bytecode generated by a Java compiler.
It would be faster to translate Java directly into a real processor’s machine language, but not as portable.

Classes, Methods, and Objects
Programs consist of instructions and data.
Classes are Java’s way of grouping particular data with the instructions that operate on them.
The compiler generates a separate class file for every class in your program.  One .java file can define anywhere from one to dozens of classes.
All the instructions in a program are enclosed in methods.
Even constructors and initializers are actually compiled into methods.
All the variables and constants in a program make up its data.
May be defined inside or outside of methods, but always inside a class.
If outside any method, may be either static or instance data.
Objects are “instances” of classes.
