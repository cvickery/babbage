Two Event Handling Strategies
• Textbook:  Have one class that implements all
Listener interfaces for all Components.
– Each callback method examines the Event object
passed to it to determine which component was the
source of the event.
• Can lead to a big switch statement in each callback method.
• This Course:  Create a separate Listener class for
every component.
– Each callback method contains logic only for a single
component, making the code easier to manage.
– Need lots of little classes.
• Above two strategies can be mixed in one
application.
