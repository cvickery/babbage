The throw Operator
• It’s an operator the way new is an operator.
– But throw doesn’t return a value; it causes abrupt
termination.
• It’s a unary operator, like new and the “negate”
form of -.
– One operand
– To the right of it
– Has to be an object of class Throwable.
– Often created with new:
 throw new Exception( “A meaningful message” );
 throw new RuntimeException( “Another m.m.” );
– (RuntimeExceptions don’t have to be declared in the throws
clause of a method.)
