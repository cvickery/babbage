Exception Management
• Exceptions and Errors
– Exceptions are things the programmer can anticipate.
• User enters letters instead of digits, for example.
– Errors are outside the programmer’s control.
• Running out of memory, for example.
• The Java model: An exception is represented as an object
that is created in a method, which can handle it itself, or
pass it back to its calling method, which can handle it
itself, or pass it back to its calling method, which …
– If it gets back to the top of the thread’s calling stack, the thread
terminates – ungracefully.
• When an exception is thrown, it causes abrupt termination
of the method that was executing.
– No more statements get executed
– No value gets returned
