Containers and Components
Containers hold things
Components do things.
All Containers are Components
java.awt.Container is a subclass of java.awt.Component
We will use Components from the javax.swing package.
JFrame objects will be our top level windows.

To Build a GUI
Establish the layout(s) to use.
Border, Flow, Grid, …
Create Container(s) and add Component(s) to them.
Swing’s JFrame has a “Content Pane” that acts as the container.  BorderLayout is default.
Establish listeners to respond to events generated by the components.
Compute the actual geometry of the GUI
pack()
Display the GUI on the screen
show() or setVisible() on the top-level window.

Event Handling Terminology
AWT
Abstract Windowing Toolkit.  The Java classes that interact with the operating system to display windows on the screen and to respond to user interactions.  First graphical operation starts a separate thread of execution that the AWT uses to respond to the user’s actions.
Event
When a user interacts with a component, the AWT thread creates an Event object (or a subclass of Event), which has instance variables giving information about the event.
What component the user interacted with
What mouse button was clicked and/or what key was pressed
Mouse coordinates, etc.

Callback Methods and Listeners
A callback method is a method that is called by the AWT thread when an event occurs.
The Event object is passed as a parameter to the callback method.
A listener is a class that has callback methods defined in it.
In order to be a listener, a class must implement the appropriate XxxListener interface.
