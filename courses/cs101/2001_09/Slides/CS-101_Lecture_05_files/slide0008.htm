•  df_1, df_2, and i have lifetimes from the moment the method is
entered to the moment it returns.
• The scope of df_1 is the entire method; the scope of df_2 and i is only
inside the loop (“block local”)
• When the assignment to df_2 is executed the second time through the
loop there will be no more references to the first object created, and it
will no longer be “alive.” Its memory can be reused.
• After the loop finishes and before the method returns, 1,000 objects
will have been created, but only the last one will still be alive, with a
single reference to it from df_1.
