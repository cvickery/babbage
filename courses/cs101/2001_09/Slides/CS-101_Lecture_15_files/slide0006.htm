HexDump Program
• Usage: java HexDump <filename(s)>
– Supply as many file names as you want after the
HexDump class name
• Displays 16 bytes per line.
– At the left is the offset into the file of the first line
– Then 16 pairs of hexadecimal digits (one pair per byte),
with embedded spaces to make them easier to read.
– At the right are the characters for any bytes that
represent printable ASCII characters.
• Non-ASCII bytes print as dots here.
• Good for looking at the “raw” contents of Java
class files, but can be used to examine any file.
