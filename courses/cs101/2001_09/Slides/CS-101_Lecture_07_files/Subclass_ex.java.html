Subclass_ex.java

The first and second calls to println() print exactly the same thing, even though they use two different types of reference variables because both variables point to the same object and thus invoke the same toString() method.

The second and third calls to println() print different things, even though they use the same reference variable, because the variable points to different types of objects and thus refers to different toString() methods. 

    public class Subclass_ex extends java.lang.Object
    {
      private String theString;
      public Subclass_ex( String str )
      {
        theString = str;
      }
      //  Override superclass' toString() method.
      public String toString()
      {
        return theString;
      }
    }

    class UseSubclass_ex
    {
      public static void main( String[] args )
      {
        Subclass_ex s1 = new Subclass_ex( "hello" );
        Object      o1 = s1;

        System.out.println( s1.toString() );
        System.out.println( o1.toString() );
        o1 = new Object();
        System.out.println( o1.toString() );
        System.exit( 0 );
      }
    }
