Object-Oriented Topics
The finalize() method.
Gets called automatically just before an object is garbage collected.
Might not ever be run!
The protected field/method access modifier.
Allows code in subclasses to access the fields or methods.
Less restrictive than private
Subclasses don’t have to be in the same package, so less restrictive than package access, too.

Abstract Classes and Interfaces
Abstract classes cannot be instantiated.
They provide some, but not all, method definitions needed to make a useful object.
Interfaces declare method signatures that a class implementing the interface must define or inherit.
Provides a “contract” – If a class implements an interface, objects of that class can be used to call the methods of the interface.
A class can extend exactly one other class, but it can implement any number of interfaces.
Interfaces solve the ambiguity problem of multiple inheritance (which of two methods with identical signatures would be inherited?) by implementing interface methods in the subclass.
