Abstract Classes and Interfaces
• Abstract classes cannot be instantiated.
– They provide some, but not all, method definitions needed to make
a useful object.
• Interfaces declare method signatures that a class
implementing the interface must define or inherit.
– Provides a “contract” – If a class implements an interface, objects
of that class can be used to call the methods of the interface.
• A class can extend exactly one other class, but it can
implement any number of interfaces.
– Interfaces solve the ambiguity problem of multiple inheritance
(which of two methods with identical signatures would be
inherited?) by implementing interface methods in the subclass.
