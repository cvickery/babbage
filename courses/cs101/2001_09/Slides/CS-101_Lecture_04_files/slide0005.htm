Static vs. Instance Methods
• Only one copy of each method is ever loaded into
memory, regardless of whether the method is
static or not.
– Executable machine language (bytecodes) never
changes once it is loaded into memory
– Parameters and local variables are allocated memory
from the stack, which makes it possible to make
independent invocations of the method’s bytecodes.
– The difference between static and instance methods is
that instance methods always receive an extra (hidden)
parameter named this, which is a reference to the object
used to invoke the method.
