Exception Handling

Chapter 14. The examples in the chapter make heavy use of GUI components, but the basic concepts are all presented there. Concepts include:
  • The @throws Javadoc tag. There should be one of these tags for each type of exception a method throws. The tag should name the exception type and tell under what circumstances the exception might be thrown.
  • The throws clause. if a method might throw any exceptions, put "throws" followed by a comma-separated list of exception types between the parameter list and the opening brace of the method definition.
  • The throw operator. Used inside a method to throw an exception. The single operand on the right must be a reference to an object of class Exception or one of its subclasses.
  • try, catch, and finally. Syntax covered in lecture and in the book. There can be multiple catch clauses; the finally clause is optional. Know the ordering of multiple catch clauses (according to class hierarchy.)
  • Be able to define and use new subclasses of Exception. (Code given in lecture.)

I/O Processing

You should be able to use the following classes and at least some of their methods (the methods used in assignments/lectures):
  • java.io.File
  • java.io.BufferedReader
  • java.io.FileInputStream
  • java.io.DataInputStream
  • java.io.IOException

Class File Assignment

Know the differences and similarities between text and binary files. Be able to read binary information, either signed or unsigned, from a class file (or other binary data file). Know the sizes of the various Java primitive data types. Understand sign extension. Be comfortable with hexadecimal representation of binary values. Know what the magic number at the beginning of class files is.
