CS-101 Exam 1 Study Guide

October 8, 1998

Note: You will be assigned a seat for the exam, so arrive early in order to have time to find your seat assignment.

The exam will be based on lectures, as well as Chapters 6 and 14 in the textbook. There will be no questions based directly on chapters 1-5, but you are expected to know that material already.

Some material that was covered in class is not covered in the textbook. In particular, the material on the [ Java Virtual Machine ]. Go to the [ Syllabus Page ] for the course to get those and additional lecture notes.

You should also be familiar with the operation of the JDK development tools, and described in the [ Development Tools ] web page.

You should be able to write class and method definitions equivalent to the sample code covered in class.

In particular you should be able to answer questions like the following:

"Write a static method named square() that receives a two-dimensional array of Objects and returns a boolean value indicating whether the array has the same number of columns in each row as the number of rows, or not."

or

"Write a static method named longest() that receives an array of Strings, and returns the longest one."

or

"Class Character provides a method with the following signature:

public static boolean isUpperCase(char ch)

Write a static method named uppity() that receives a two-dimensional array of char and returns a double between 0 and 1.0 that tells what proportion of the arrray contains upper-case characters."

You should be able to write code that uses a StringTokenizer to process lines read using a BufferedReader. You should be able to use Integer.parseInt(), and you should be able to manage any exceptions it might throw.

For example:

"Write an application that reads lines from the user. The user should type a list of integer numbers, separated by spaces. The program should print an error message if the user types anything but numbers, but should continue processing the remainder of the user’s input even if there is an error. When the user types ^Z, the program prints the valid numbers from each line, with each line preceeded by its sum followed by a colon.

Sample session. Computer output is boldface, user’s input is monospaced.

Enter a line: 123 456

Enter a line: 987 oops! 12 19

Error: "oops!" is not a number.

Enter a line: ^Z

Your lines and their sums were:

579: 123 456

1018: 987 12 19
